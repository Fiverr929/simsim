The Module Map

Core (build first)
📋 Listing Creator
Upload image → AI generates title, description, 13 tags. Default details auto-fill. Publish to Etsy.
Research
🔍 Keyword Research
Find high-traffic tags and titles before creating a listing. Attach keywords to a listing draft.
Operations
📦 Orders
Pull all Etsy orders. Buyer name, address, items bought, fulfilment status.
Money
💰 Finance
Revenue, Etsy fees, profit per listing. Monthly summaries. Derived from Orders data.
Fulfilment
🖨️ Printify
Link listings to Printify products. Auto-push orders to Printify for fulfilment.
Config
⚙️ Shop Defaults
Set once: Who Made, Type, Taxonomy, Return Policy. Pre-fills every new listing automatically.

Pick the structure that feels right

A

Dedicated Etsy section in the sidebar

The workspace sidebar gets an "Etsy" section alongside the generic bases. Each module (Listings, Orders, Finance, Keywords, Printify) is its own page inside that section. Clean, purpose-built, like a mini-app inside the workspace.

Recommended
Best UX. Each module gets its own layout. Scales cleanly as you add Amazon later.
B

Etsy as a smart base with guided table creation

Instead of a seed route, a "New Base" wizard lets you pick "Etsy Store". It creates the Listings, Orders, and Customers tables automatically with the right fields. The automation bar stays in RecordModal as-is. Keyword research and Finance are views within those tables.

Trade-off
Stays within the current workspace paradigm. Harder to build dedicated UIs like Keyword Research or Finance dashboards.
