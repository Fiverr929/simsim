Etsy Seller Workspace — Full Module Map

7 modules. Pick the structure that fits best.

📋
Create
Listing Creator
Upload image → AI writes title, description, 13 tags → review → publish to Etsy. Shop Defaults auto-fill repeat fields so you only fill what changes.
🟢
Active Listings
Sync all live Etsy listings. View stats (views, favourites, sales). Bulk edit title/tags/price. Renew expiring listings. Deactivate.
🔍
Keyword Research
Research tags and titles. Attach winning keywords to a listing draft before publishing.
📦
Orders
Pull all Etsy orders. Buyer name, address, items, fulfilment status. Link to listings.
💰
Finance
Revenue, Etsy fees, profit per listing. Monthly summaries. Derived from Orders.
🖨️
Printify Sync
Link Active Listings to Printify products. Auto-push new orders to Printify for fulfilment.
⚙️
Shop Defaults
Set once: Who Made, Type, Taxonomy ID, Return Policy, Shop Section. Pre-fills every new listing.
How the data connects
Listing Creator → publishes to Etsy → appears in Active Listings
Active Listings ↔ linked to Printify products
Orders → auto-sent to Printify for fulfilment → feeds Finance
Shop Defaults → pre-fills Listing Creator fields
Keyword Research → attached to listings in Listing Creator

How should this live in the app?

A

Dedicated Etsy section in the sidebar

The workspace gets an "Etsy" nav section. Each module is its own page with its own layout — the Listing Creator is a form, Orders is a table, Finance is a dashboard, etc. Clean and purpose-built.

✓ Recommended — each module gets the right UI, scales to Amazon later
B

Etsy as smart base templates in the existing workspace

A "New Base" wizard creates the right tables automatically. All modules live as tables/views inside the workspace. No new nav section needed.

⚠ Harder to build proper UIs for Finance, Keywords, and Printify sync
